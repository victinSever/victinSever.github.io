索引操作

  • 优点：提高数据库查询效率，降低数据库IO，降低数据排序成本，降低CPU消耗

  • 缺点：占用磁盘空间，增加了索引数据，空间插入和修改需要更新索引

  • 语法

    • 创建：db.集合名.createIndex(待创建索引的列,[,额外选项])
      • 1升序， -1降序
    • 删除全部：db.集合名.dropIndexes()
    • 删除指定：db.集合名.dropIndex(索引名)
    • 查看索引：db.集合名.getIndexes()
    • 分析索引：db.集合名.explain(“executionStatus”)

  • 练习

    • 创建唯一索引

  • 选择规则

    • 为常用条件、排序、分组的字段简历索引
    • 选择唯一性索引
    • 选择较小的数据列，为长字符串使用前缀索引

使用权限机制开启账号机制

  • 开启验证模式

    • 添加超级管理员

      • ```js mongodb
        use admin

        db.createUser({
        “user”: “admin”,
        “pwd”: “123456”,
        “roles”: [{
        role: “root”,
        db: “admin”
        }]
        })

    • 重新安装需要输入账密的服务

      • ```js mongodb

      • ```js mongodb
        mongo或者mongosh

    • 添加用户权限

      • ```js mongodb
        //在管理员权限下，进入新的数据库

        use shop

        //shop1 只读权限
        db.createUser({
        “user”: “shop1”,
        “pwd”: “123456”,

        role: “read”,
        db: “shop”
        }]
        })
        //shop2 可读可写权限
        db.createUser({
        “user”: “shop2”,
